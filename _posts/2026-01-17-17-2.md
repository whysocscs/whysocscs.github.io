<p data-ke-size="size16"> </p>
<p data-ke-size="size16">막상 바이브 코딩을 해보니까 여러 가지 문제점이 발생했다. 처음에는 OpenAI를 활용해서 GPT를 활용하려 했지만, 일단은 무료로 하고 싶어서 Llama로 넘어가는 점, 그러다 보니 처음 명세서랑 달라서 여러 가지 문제점..</p>
<p data-ke-size="size16"> </p>
<p data-ke-size="size16">밑에 글은 GPT를 이용해서 좀 다듬었다. 이슈 트러블링을 하기 위해서 남겨놓은 노션이 너무 이상한 이유와 글들도 많아서 딱 어떤 이슈가 있었고, 어떻게 해결했는지만 남긴ㄷ..</p>
<p data-ke-size="size16"> </p>
<p data-ke-size="size16"> </p>
<h3 data-end="226" data-ke-size="size23" data-start="187">무료 요약 모델 도입 + 노션 연동 삽질기 + 자잘한 버그 정리</h3>
<p data-end="307" data-ke-size="size16" data-start="228">지난 글에서는 “최신 보안 기사들을 모아서 요약하고, 노션에 쌓아두는 파이프라인”을 왜 만들게 됐는지랑, 대략 어떤 식으로 시작했는지 적었다.</p>
<p data-end="427" data-ke-size="size16" data-start="309">이번 글은 그 다음 단계다.<br/>만들다 보면 늘 그렇듯이, 기능 구현보다 <b>연동/환경/사소한 버그</b>에서 시간이 더 갈렸다. 그래서 이번에는 내가 겪은 삽질을 그대로 정리해두려고 한다. (나중에 또 겪을까 봐)</p>
<hr data-end="432" data-ke-style="style1" data-start="429">
<h2 data-end="480" data-ke-size="size26" data-start="434">1) 요약 모델 교체: 유료 GPT → 무료 Llama 3로 갈아탄 이유</h2>
<p data-end="602" data-ke-size="size16" data-start="482">처음 프로젝트는 OpenAI GPT 모델을 기준으로 되어 있었다. 문제는 운영을 조금만 해도 비용이 발생한다는 점이었다.<br/>마음껏 돌릴 수 있는 구조로 만들려면, <b>무료이면서도 성능이 괜찮은 대안</b>이 필요했다.</p>
<p data-end="634" data-ke-size="size16" data-start="604">그래서 선택한 게 <b>Groq의 Llama 3</b>였다.</p>
<ul data-end="737" data-ke-list-type="disc" data-start="636" style="list-style-type: disc;">
<li data-end="737" data-start="636"><b>왜 Groq?</b>
<ul data-end="737" data-ke-list-type="disc" data-start="652" style="list-style-type: disc;">
<li data-end="668" data-start="652">응답 속도가 진짜 빠른 편</li>
<li data-end="686" data-start="671">무료 사용량이 넉넉한 편</li>
<li data-end="737" data-start="689">무엇보다 <b>OpenAI API 형식과 호환</b>이라 코드 변경을 최소화할 수 있었다</li>
</ul>
</li>
</ul>
<p data-end="786" data-ke-size="size16" data-start="739">여기서 끝이 아니라, 나중에 또 다른 모델로 쉽게 바꾸고 싶어서 코드 구조도 손봤다.</p>
<ul data-end="888" data-ke-list-type="disc" data-start="788" style="list-style-type: disc;">
<li data-end="822" data-start="788">기존: 특정 모델에 묶인 OpenAISummarizer</li>
<li data-end="888" data-start="823">개선: <b>OpenAI 호환 API면 다 연결 가능한</b> ChatCompletionSummarizer (범용)</li>
</ul>
<hr data-end="1018" data-ke-style="style1" data-start="1015"/>
<h2 data-end="1062" data-ke-size="size26" data-start="1020">2) 미스터리한 API 키 오류: .env에 분명 있는데 왜 못 읽지?</h2>
<p data-end="1140" data-ke-size="size16" data-start="1064">Notion이랑 Groq API 키는 .env에 넣어두고 돌렸다.<br/>근데 프로그램이 계속 <b>API 키가 없는 것처럼</b> 동작했다.</p>
<ul data-end="1202" data-ke-list-type="disc" data-start="1142" style="list-style-type: disc;">
<li data-end="1165" data-start="1142">Notion에는 아무것도 저장이 안 됨</li>
<li data-end="1202" data-start="1166">요약도 제대로 안 되고, 저품질 <b>Dummy 요약</b>만 뱉음</li>
</ul>
<p data-end="1242" data-ke-size="size16" data-start="1204">처음에는 키를 잘못 넣었나 싶었는데, .env에는 분명히 있었다.</p>
<p data-end="1274" data-ke-size="size16" data-start="1244">그래서 원인을 쪼개려고 진단 스크립트를 따로 만들었다.</p>
<ul data-end="1313" data-ke-list-type="disc" data-start="1276" style="list-style-type: disc;">
<li data-end="1295" data-start="1276">check_notion.py</li>
<li data-end="1313" data-start="1296">check_groq.py</li>
</ul>
<p data-end="1382" data-ke-size="size16" data-start="1315">이 스크립트들은 정상적으로 키를 읽고 통신도 성공했다.<br/>근데 <b>메인 프로그램만</b> 키를 못 읽는 기현상이 이어졌다.</p>
<p data-end="1397" data-ke-size="size16" data-start="1384">원인은 의외로 단순했다.</p>
<blockquote data-end="1428" data-ke-style="style1" data-start="1399">
<p data-end="1428" data-ke-size="size16" data-start="1401">메인 프로그램에 .env 로드 코드가 없었다.</p>
</blockquote>
<p data-end="1455" data-ke-size="size16" data-start="1430">해결은 python-dotenv로 끝났다.</p>
<ul data-end="1538" data-ke-list-type="disc" data-start="1457" style="list-style-type: disc;">
<li data-end="1497" data-start="1457">requirements.txt에 python-dotenv 추가</li>
<li data-end="1538" data-start="1498">run.py 시작 부분에 load_dotenv() 한 줄 추가</li>
</ul>
<hr data-end="1603" data-ke-style="style1" data-start="1600"/>
<h2 data-end="1637" data-ke-size="size26" data-start="1605">3) Notion 연동 삽질기: 오류가 3연타로 터짐</h2>
<p data-end="1720" data-ke-size="size16" data-start="1639">환경 변수 문제를 해결했는데도 Notion은 또 다른 이유로 실패했다.<br/>여기서는 check_notion.py가 거의 핵심 디버깅 도구였다.</p>
<h3 data-end="1752" data-ke-size="size23" data-start="1722">(1) 1차 오류: 400 Bad Request</h3>
<ul data-end="1834" data-ke-list-type="disc" data-start="1753" style="list-style-type: disc;">
<li data-end="1792" data-start="1753">내용: NOTION_DATABASE_ID가 유효하지 않다고 나옴</li>
<li data-end="1834" data-start="1793">원인: 데이터베이스 ID가 아니라 <b>페이지 ID</b>를 복사한 상태였다</li>
</ul>
<p data-end="1881" data-ke-size="size16" data-start="1836">Notion URL 구조를 다시 보고, DB ID를 정확히 추출해서 해결했다.</p>
<h3 data-end="1908" data-ke-size="size23" data-start="1883">(2) 2차 오류: IndexError</h3>
<p data-end="1942" data-ke-size="size16" data-start="1909">DB ID를 고치니까 이번엔 IndexError가 떴다.</p>
<ul data-end="2021" data-ke-list-type="disc" data-start="1944" style="list-style-type: disc;">
<li data-end="1992" data-start="1944">원인: 데이터베이스 <b>제목(Title)이 비어 있는 경우</b> 진단 스크립트가 터짐</li>
<li data-end="2021" data-start="1993">해결: 노션에서 DB 제목을 입력 → 바로 해결</li>
</ul>
<p data-end="2083" data-ke-size="size16" data-start="2023">이때 깨달아서, check_notion.py도 보강했다.<br/>(제목 없는 DB여도 오류 없이 처리하도록)</p>
<h3 data-end="2115" data-ke-size="size23" data-start="2085">(3) 3차 오류: Schema Mismatch</h3>
<p data-end="2130" data-ke-size="size16" data-start="2116">마지막 관문은 스키마였다.</p>
<ul data-end="2179" data-ke-list-type="disc" data-start="2132" style="list-style-type: disc;">
<li data-end="2179" data-start="2132">원인: Notion DB의 속성(컬럼)들이 코드에서 기대하는 타입/이름이랑 불일치</li>
</ul>
<p data-end="2234" data-ke-size="size16" data-start="2181">그래서 필요한 속성들을 <b>이름/타입까지 정확히 맞춰</b> 세팅했고, 그제야 정상 업로드가 됐다.</p>
<p data-end="2274" data-ke-size="size16" data-start="2236">예를 들어, 보통 이런 식으로 “정확한 속성명 + 타입”이 중요했다.</p>
<ul data-end="2412" data-ke-list-type="disc" data-start="2276" style="list-style-type: disc;">
<li data-end="2293" data-start="2276">Title : Title</li>
<li data-end="2333" data-start="2294">Source : Select 또는 URL(프로젝트 설계에 따라)</li>
<li data-end="2366" data-start="2334">Risk : Select / Multi-select</li>
<li data-end="2412" data-start="2367">Date : Date<br/>(이런 식으로 코드와 DB가 1:1로 맞아야 함)</li>
</ul>
